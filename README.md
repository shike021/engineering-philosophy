# Engineering Philosophy

**在复杂系统中做出正确取舍的工程思考系列**

作者：Eric
日期：2026.04

## 关于本项目

本仓库收录了一系列工程实践中的思考与总结，主题涵盖迭代、取舍、确定性、创新判断、AI 影响、权力与责任等。

**项目目的**：通过本人在真实项目中的观察与反思，提炼一套可操作、可传承的工程决策框架，帮助团队和后续项目少走弯路、减少系统性错误，让工程交付更确定、可控、可持续。

> **重要声明**：本文档及所有文章内容均为作者个人工程实践思考与观点，仅代表个人立场，不代表任何公司或组织的官方观点。欢迎讨论与交流，但请勿用于商业决策而未经过独立验证。

## 8 条核心工程哲学原则

1. 迭代能力 > 初始完美
2. 价值判断 > 难度追求
3. 确定性边界不可打破
4. 有所为，有所不为
5. 复杂性放在正确的位置
6. AI 是加速器，不是替身
7. 权力与责任必须 1:1 对齐
8. 工程本质是持续正确的取舍
